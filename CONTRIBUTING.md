# 贡献指南

感谢你考虑为悬浮时间项目做出贡献！

## 如何贡献

### 报告 Bug

如果你发现 Bug，请创建 Issue 并提供以下信息：

- Bug 描述
- 复现步骤
- 预期行为
- 实际行为
- 环境信息（设备型号、系统版本、应用版本）
- 截图或日志（如有）

### 建议新功能

欢迎提出新功能建议：

- 功能描述
- 使用场景
- 预期效果
- 可能的实现方案（可选）

### 提交代码

1. Fork 本仓库
2. 创建特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启 Pull Request

### 代码规范

- 使用 TypeScript 编写代码
- 遵循 ESLint 规则
- 添加必要的注释
- 保持代码简洁清晰

### 提交信息规范

使用语义化提交信息：

- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建或工具相关

示例：
```
feat: 添加夜间模式支持
fix: 修复时间同步偏移计算错误
docs: 更新部署指南
```

## 开发环境设置

请参考 [README.md](README.md) 的"快速开始"部分。

## 测试

在提交 PR 前，请确保：

- [ ] 代码通过 ESLint 检查
- [ ] 在 Android 和 iOS 上测试通过
- [ ] 没有引入新的警告或错误

## 行为准则

- 尊重所有贡献者
- 接受建设性批评
- 专注于对项目最有利的事情
- 对社区其他成员表现出同理心

## 问题？

如有任何问题，请：
- 查看现有 Issues
- 创建新 Issue
- 联系维护者

再次感谢你的贡献！
